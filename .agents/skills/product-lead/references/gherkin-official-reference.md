# Gherkin Official Reference

## Purpose

Keep parser-level syntax decisions anchored to official sources.

## Canonical Sources

- Cucumber Gherkin reference (primary): https://cucumber.io/docs/gherkin/reference
- Cucumber docs landing page: https://cucumber.io/docs/
- Gherkin parser repository: https://github.com/cucumber/gherkin

## Usage Guidance

- For keyword validity and grammar disputes (`Rule`, `Background`, `Scenario Outline`, `Examples`, aliases), treat the official Cucumber Gherkin reference as the source of truth.
- For team writing style and repository conventions, use `gherkin-best-practices.md`.
- Runtime runner concerns (`playwright-bdd`, step matching, `bddgen`) are owned by `tech-lead` documentation.
- If local guidance conflicts with official syntax, update the local guidance to align with official syntax.
